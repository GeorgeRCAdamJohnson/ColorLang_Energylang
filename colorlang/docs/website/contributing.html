Contributing

Thank you for your interest in contributing to ColorLang! This document outlines the process for making contributions, including coding standards, submitting changes, and collaborating effectively.

How to Contribute

  • Report Issues: Use the tickets folder to log recurring issues.
  • Propose Changes: Discuss your ideas with the team before making significant changes.
  • Submit Pull Requests: Follow the coding standards and submit clear descriptions of your changes.

Coding Standards

  • General Guidelines: Write clean, readable, and well-documented code.
  • Python Standards: Use black for code formatting and follow PEP 8 guidelines.
  • ColorLang Standards: Ensure all new functionality is implemented in ColorLang where possible.
