Shared Memory

Shared memory in ColorLang facilitates communication between the Virtual Machine (VM) and host applications. It is a critical component for enabling real-time interactions, such as AI-driven behavior and rendering updates.

Key Components

  • Tilemap: Represents the game or application environment as a 2D grid.
  • Agent State: Tracks the state of AI agents, including position, emotional state, and performance metrics.
  • Cognition Strip: Represents the decision-making process of the AI agent.

Data Flow

  1. The VM updates shared memory during program execution.
  2. Changes are reflected in the tilemap, agent state, and cognition strip.
  3. The host application can modify shared memory to provide input.
