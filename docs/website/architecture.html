Architecture

ColorLang is a machine-native programming language designed to encode programs as HSV pixel grids. The system architecture consists of several key components that work together to parse, execute, and communicate ColorLang programs.

High-Level Components

  • Color Parser: Converts HSV pixel grids into executable instructions.
  • Virtual Machine (VM): Executes parsed ColorLang programs.
  • Shared Memory: Facilitates communication between the VM and host runtime.
  • Host Applications: Integrate with the VM to showcase ColorLang's capabilities.

Data Flow

  1. The user provides a ColorLang program as an image file.
  2. The Color Parser decodes the image into instructions.
  3. The VM executes the instructions, updating registers and shared memory.
  4. The host application renders the results or communicates them to other systems.
